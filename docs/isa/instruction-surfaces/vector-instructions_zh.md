# 向量指令集

`pto.v*` 指令集直接暴露向量流水线，用于对 lane 级寄存器、谓词和向量可见 UB 状态做细粒度控制。

## 指令集概览

| 类别 | 说明 |
| --- | --- |
| 向量加载存储 | UB 与向量寄存器之间的数据搬运 |
| 谓词与物化 | 广播、复制和谓词相关结果 |
| 一元向量操作 | 单输入 lane 级操作 |
| 二元向量操作 | 双输入 lane 级操作 |
| 向量-标量操作 | 向量与标量混合操作 |
| 转换操作 | 类型转换和索引生成 |
| 归约操作 | 跨 lane 的归约 |
| 比较与选择 | 比较、谓词生成和按 mask 选择 |
| 数据重排 | pack、permute、interleave、slide 等 |
| SFU 与 DSA | 特殊函数、融合形式和特定领域操作 |

## 输入

向量指令集常见输入包括：

- 向量寄存器 `!pto.vreg<NxT>`
- 标量寄存器或立即数
- 谓词 mask
- UB 指针
- 分布、舍入或对齐相关属性

## 输出

向量指令会产生：

- 向量寄存器结果
- 标量结果
- 谓词结果
- 通过 vector store 写回的 UB 数据

## 约束

- 向量宽度由元素类型决定，不是任意选择。
- 谓词宽度必须与目标向量宽度匹配。
- 某些对齐和非对齐 store 形式受 profile 限制。
- 向量指令集没有 tile 层的 valid region 语义。

## 不允许的情形

- 使用与向量宽度不匹配的谓词
- 在不支持的 profile 上使用 A5 专属向量形式
- 把未文档化的 masked lane 行为当作固定契约
